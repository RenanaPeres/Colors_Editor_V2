# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Nature

This repository contains **build artifacts only** — there are no source files, package.json, or build configuration committed. The compiled output was generated by Vite (evidenced by content-hashed asset filenames). The original source is developed separately.

Files present:
- `index.html` — app shell with hardcoded asset references
- `assets/index-[hash].js` — compiled/minified JavaScript bundle
- `assets/index-[hash].css` — compiled/minified CSS bundle
- `gallery/fgGallery.jpg.png` and `gallery/bgGallery.jpg.png` — sprite sheets for image selection

To deploy updates, replace the asset files and update the `<script src>` and `<link href>` references in `index.html` to match the new hashed filenames.

## Application Architecture

**Colors Editor V2** is a two-layer photo compositor running entirely in the browser via HTML5 Canvas.

### Layout
- Fixed 500px **sidebar** (left) + fluid **workspace** canvas (right)
- Sidebar has two independent layer sections (Foreground on top, Background below) and a footer with action buttons

### Layer Model
Each layer (foreground `fg`, background `bg`) has independent sliders:
- **Color**: Hue (`hue-rotate`, 0–360°), Saturation (`saturate`, 0–200%)
- **Details**: Brightness (0–200%), Contrast (0–200%), Blur (0–20px, step 0.5)

### Rendering Pipeline
1. Background image drawn onto canvas with its CSS filter string applied via `ctx.filter`
2. Foreground image drawn centered on top with its own filter string
3. Filter strings use the format: `hue-rotate(Xdeg) saturate(X%) brightness(X%) contrast(X%) blur(Xpx)`

### State & History
- Two image objects and two filter-state objects are kept in memory
- A history stack (array + pointer) enables undo/redo
- Reset Filters button returns all sliders to defaults without clearing image selections

### Gallery
Images are selected from sprite sheets (`fgGallery.jpg.png`, `bgGallery.jpg.png`). Click events on the gallery `<div>` map pixel offset to sprite frame index.

### Export
Download button exports two files:
1. A composited PNG (new off-screen canvas with filters baked in via `drawImage`)
2. A JSON state report as a `.txt` file
